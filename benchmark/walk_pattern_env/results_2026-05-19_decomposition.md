## Tech 4 phase decomposition

`walk_lazy_next_dfs(parse(str, XML.LazyNode), Ref(0))` decomposes into
three measurable phases. Each phase measured by subtraction (phase B
minus phase A, etc.).

| Phase | Time | Memory | Notes |
|---|---|---|---|
| Parse ‚Äî `parse(str, XML.LazyNode)` | 14 ms | 21 MiB | 8 allocs (constant) |
| Cursor advance ‚Äî `XML.next!()` loop, no value extraction | 26 ms | 79 MiB | ~1.3M allocs (from `Union{Nothing, LazyNode}` return boundary) |
| Value extraction ‚Äî `XML.value()` on Text/CData nodes | 17 ms | 23 MiB | ~600k Strings (one per Text/CData node) |
| **Total (Tech 4)** | **57 ms** | **123 MiB** | **1.9M** |

Walk-only (= cursor advance + value extraction) = 43 ms / 102 MiB.

Notable: only **1** `LazyNode` wrapper is allocated by tech 4 across
the entire 100k-Placemark walk (allocated upfront, mutated in place by
`XML.next!()` from that point on). The 1.9M total allocs come from
internal call-boundary effects (Union return on `next!()`) and from
String materialization in `XML.value()` ‚Äî *not* from wrapper allocation
per child.

## Finding: v0.4's `parse(str, XML.LazyNode)` is O(1)

Measured separately in the v0.4 env: `parse(str, XML.LazyNode)` returns
in **0.00 ms / 0 MiB / 0 allocs** (below BenchmarkTools' rounding
threshold) ‚Äî confirming v0.4's `LazyNode` is a fully lazy wrapper that
defers all tokenization to walk-time. Consequence: tech 6's "walk-only"
figure (293 ms) **is** its parse-plus-walk total, directly comparable
to tech 4's parse-plus-walk total of 57 ms.

This contrasts with v0.3.8 + #59, where `parse(str, XML.LazyNode)`
allocates ~21 MiB (the LazyNode + token state) and takes ~14 ms.

## Reproduction

The 2026-05-19 measurements are produced by `decompose_techniques.jl`
in this directory, run once per env:

```sh
# Tech 4 decomposition (v0.3.8 + PR #58 + PR #59 env)
julia --project=benchmark/walk_pattern_env \
    benchmark/walk_pattern_env/decompose_techniques.jl \
    /Users/.../FastKML.jl/dev/XML.jl

# Tech 3 + Tech 6 decomposition (v0.4 env)
julia --project=benchmark/walk_pattern_env \
    benchmark/walk_pattern_env/decompose_techniques.jl \
    /Users/.../FastKML.jl/dev/XML.jl-v0.4
```

Each invocation activates a temporary Pkg env with `Pkg.develop` against
the supplied XML.jl checkout, so the bench can be re-run without
disturbing the host project state.
